# План рефакторинга money-stat

## Задачи

1. Инициализация базы данных — вынести в отдельный этап
2. Команды Cobra — вынести в отдельные пакеты

## Анализ текущей структуры

- Основной код находится в `main.go`
- Команды Cobra определены в пакете `cmd`
- Инициализация базы данных происходит в `internal/app/db.go` через `NewDB()`
- Используется GORM для работы с SQLite

## Предлагаемое решение

### 1. Инициализация базы данных

Вынести инициализацию базы данных в отдельный пакет `internal/dbinit` с возможностью явного вызова инициализации.

### 2. Команды Cobra

Создать отдельный пакет для каждой команды в директории `cmd/`:

- `cmd/accounts`
- `cmd/capital`
- `cmd/months`
- `cmd/sync`
- `cmd/year`

## Этапы реализации

### Этап 1: Создание пакета инициализации базы данных

1. Создать директорию `internal/dbinit`
2. Создать файл `internal/dbinit/dbinit.go` с функцией инициализации базы данных
3. Модифицировать `internal/app/db.go` для использования новой структуры инициализации

### Этап 2: Реорганизация команд Cobra

1. Для каждой команды из пакета `cmd` создать отдельную директорию
2. Переместить соответствующий код команды в новую директорию
3. Обновить импорты в `main.go` и других файлах

### Этап 3: Обновление main.go

1. Обновить импорты для новых путей команд
2. Убедиться, что все команды корректно регистрируются

## Проверка изменений

1. Запустить `go build` для проверки сборки
2. Выполнить основные команды приложения для проверки работоспособности
3. Убедиться, что инициализация базы данных работает корректно

## Критические файлы

- `main.go`
- `internal/app/db.go`
- `cmd/*.go`
- `internal/dbinit/dbinit.go` (новый файл)
