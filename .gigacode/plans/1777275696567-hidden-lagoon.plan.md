# План: Кэширование результата GetCapital

## Контекст

Функция `GetCapital` в `internal/usecase/capital.go` может быть медленной при большом объеме данных, так как каждый
вызов загружает все транзакции и пересчитывает баланс. Предлагается добавить кэширование результата на определенное
время, недействительным при появлении новых транзакций.

## Цель

Ускорить повторные вызовы `GetCapital` за счет кэширования результата с TTL и инвалидацией при изменении данных.

## Исследование

- `GetCapital` находится в `internal/usecase/capital.go`
- Проект использует Go 1.22, зависимости в `go.mod` не включают bigcache
- В проекте нет централизованного кэша или механизма подписки на изменения данных

## Варианты реализации

1. **bigcache** - производительный кэш, но требует добавления зависимости
2. **sync.Map + ручной TTL** - использует стандартную библиотеку, проще и легче

## Рекомендуемое решение

Использовать `sync.Map` с ручным TTL, так как:

- Не требует новых зависимостей
- Достаточно для кэширования одного значения
- Соответствует простоте проекта

## План реализации

1. Добавить в структуру `Capital` поля для хранения кэшированного значения и времени его обновления
2. Реализовать проверку актуальности кэша (например, TTL 5 минут)
3. Модифицировать `GetCapital` для использования кэша при его наличии
4. Обеспечить потокобезопасность через мьютексы или атомарные операции

## Файлы для изменения

- `internal/usecase/capital.go`

## Верификация

1. Написать тест, проверяющий, что при повторном вызове `GetCapital` в течение TTL возвращается кэшированное значение
2. Убедиться, что после истечения TTL или при изменении данных (в будущем) кэш инвалидируется
3. Проверить отсутствие гонок при параллельных вызовах

## Следующие шаги

Обсудить с пользователем выбор реализации и уточнить параметры TTL.
